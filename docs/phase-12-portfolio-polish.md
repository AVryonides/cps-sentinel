# Phase 12: portfolio and GitHub polish

Phase 12 turns the working prototype into a clearer public portfolio project. The goal is not to
add a new algorithmic subsystem; it is to make the repository, dashboard, and CV material easier
to understand for a recruiter, interviewer, or engineering reviewer.

## Changes

- Removed the visible development-phase badge from the dashboard header.
- Increased spacing in the SWaT anomaly figure so subplot titles, legends, and traces breathe
  more cleanly.
- Added a project brief that explains the problem, implementation, results, and boundaries.
- Added a CV-ready summary with bullet options and interview talking points.
- Updated the README with CI/Python/license badges, a concise value proposition, validation
  snapshot, repository map, and links to the project brief and CV summary.

## Acceptance criteria

- The dashboard no longer displays an outdated phase label in the top-right header.
- The SWaT anomaly report has clearer title/legend spacing in both standalone HTML and the
  NiceGUI dashboard.
- A GitHub visitor can understand the project purpose, architecture, results, demo command, and
  data policy from the README and linked docs.
- CV material is available without exposing restricted raw dataset content.
